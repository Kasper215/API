# Веб-сервис для автоматизации приёмной кампании университета

## 1. Название и основное назначение
**Название:** Веб-сервис «Приёмная кампания»
**Назначение:** Автоматизация процесса подачи, обработки и подтверждения заявок абитуриентов на поступление в университет.

## 2. Краткое описание решаемой задачи и целевой аудитории
**Задача:** Система позволяет абитуриентам подавать заявки на две специальности, автоматически рассчитывает их итоговые баллы на основе ЕГЭ и аттестата, сопоставляет баллы с проходными и выдает результат с дальнейшим приглашением к зачислению.
**Целевая аудитория:**
- **Абитуриенты**, желающие подать документы в университет дистанционно.
- **Сотрудники приемной комиссии** для автоматизированного учета поступающих и быстрого подведения итогов.

## 3. Перечень основных функций
1. **Получение списка специальностей (`GET /api/specialties`)** — Возвращает актуальный перечень направлений подготовки и их проходные баллы.
2. **Получение списка абитуриентов (`GET /api/applicants`)** — Возвращает список зарегистрированных абитуриентов с их статусом ("абитуриент" или "студент") и рассчитанным итоговым баллом.
3. **Подача заявки (`POST /api/applications`)** — Принимает заявку на поступление (с выбором двух специальностей), рассчитывает итоговый балл и определяет результат поступления (сценарии: не прошел, прошел на одну, прошел на обе).
4. **Подтверждение зачисления (`POST /api/applications/{id}/confirm`)** — Позволяет абитуриенту согласиться на зачисление (статус меняется на "студент") или отказаться от зачисления.

## 4. Архитектурные особенности
- **Фреймворк:** ASP.NET Core Web API (C#)
- **Архитектура:** Многослойная (Models, Services, Controllers), RESTful API.
- **Хранение данных:** In-Memory (статические коллекции в оперативной памяти) для быстрой демонстрации работы бизнес-логики без необходимости настраивать СУБД.
- **Документирование API:** Swagger UI (Swashbuckle.AspNetCore) для автогенерации интерактивной документации по эндпоинтам с комментариями в формате XML.

## 5. Инструкции по развертыванию и запуску
1. Установите [.NET SDK](https://dotnet.microsoft.com/download) актуальной версии.
2. Распакуйте исходный код проекта.
3. Откройте терминал (или командную строку) в папке проекта (где находится файл `WebApplication1.csproj`).
4. Выполните команду:
   ```bash
   dotnet run
   ```
5. После запуска консоль выведет адрес (например, `http://localhost:5192`). Перейдите по адресу `http://localhost:<порт>/swagger`, чтобы открыть панель интерактивного тестирования Swagger UI.
